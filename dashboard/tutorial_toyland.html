01
Stage · Foundation

Analyze the data.

Get Claude to write your first analysis script

Before we build anything smart, we look at the data. Run a simulation. Download the excel files. Hand them to Claude. Ask for insights.

You'll set up Claude Cowork on your machine, point it at a folder of excel files, drop in our analysis prompt, and watch it generate Python code that aggregates 60 months of sales, purchases, and stock data into clean insights.

BizBotBash backend ──▶ sim_excel/*.xlsx ──▶ ~/toyland-data/ ──▶ Claude Cowork ──▶ insights + charts
  1. Run a simulation on the dashboard
    Open the dashboard, pick Bizbotbash Champion, set duration to 60 months, click Run Simulation. This calls our BizBotBash backend and produces a folder of excel files (one per calendar month).
  2. Download the excel bundle
    After the run finishes, click the Download Excel link in the dashboard (or grab it here: /toyland/download-excel). You'll get a zip with initial_state.xlsx and month_YYYY-MM.xlsx files.
  3. Set up a folder
    Create ~/toyland-data/ on your machine. Unzip the excel files into it. The folder is now your "data warehouse."
  4. Install Claude Cowork
    Grab it from claude.ai/download and point it at ~/toyland-data/. Cowork gives Claude filesystem access so it can read your excel files and write Python scripts.
  5. Drop in the analysis prompt
    Download toyland-analysis.md and save it to the folder. Open Claude Cowork and say: "Read toyland-analysis.md and do what it says."
  6. Let Claude cook
    Claude will build aggregate.py (reads all excel files into one dataframe), analysis.py (computes KPIs, trends, top movers), and charts.py (matplotlib plots). Run them. Read the output. Ask follow-up questions.
Tip   Ask Claude: "Which products had the highest revenue variance? What were the worst months for stockouts? Show me a heatmap of sales by location × category." — specifics unlock specifics.

02
Stage · Application

Build a real ERP.

Dashboard app that commands Excel

Reading data is passive. Writing data turns a dashboard into an ERP. In Stage 02 you extend what Claude built into a full app that can issue purchase orders, transfer stock, and rearrange shelves — and have those decisions actually change the excel files.

You'll use Claude Cowork to build a local web dashboard (Flask, Streamlit, whatever) on top of your excel folder. When you click "issue PO" in your app, the underlying xlsx updates. When you query a report, it reads the latest state.

┌─── your ERP UI ───┐ ┌─── BizBotBash backend ───┐ │ · New PO │ │ · Daily sales engine │ YOU ──▶ · Stock transfer │ ──xlsx─▶│ · PO / transfer execution │ │ · Shelf rearrange │ │ · Monthly excel export │ └────────────────────┘ └────────────────────────────┘
  1. Ask Claude to build a read/write layer
    Prompt: "Extend aggregate.py so it can also WRITE. Add functions to append a new PO row, mark a transfer, update a shelf layout — each one modifies the relevant xlsx file and saves it back."
  2. Add a small local web UI
    Prompt: "Build a Flask app with pages: Products, Purchase Orders, Transfers, Shelf Map. Each page has a form that calls the write functions from step 1." Claude will scaffold this in minutes.
  3. Wire the UI to BizBotBash for simulation
    When you click "Advance one month," your app calls our backend at /run or /run-bot, waits for the result, and re-exports the updated excel files into your folder. Everything stays in sync.
  4. Style it like BizBotBash
    Steal the black + yellow theme from this site. Claude will happily turn your Flask templates into a matching aesthetic. Looks matter when you're demoing to a boss.
Reality check   Excel is a terrible transactional database. Fine for this tutorial. Don't ship it to a real business. When you're ready for production, replace xlsx with Postgres and keep the same interface.
// End of Stage 02
You have a working ERP that reads from and writes to your excel folder. You can operate ToyLand by clicking buttons instead of writing Python. Every decision you make modifies the data. You just built a small business application.
03
Stage · Copilot

Add an AI copilot.

Bot recommends. Human approves.

Your ERP works — but you're still the brain. Stage 03 plugs in an AI bot that watches the state and suggests the next moves. Not executing yet. Just recommending.

This is the "copilot" pattern: AI proposes, human disposes. You see each recommendation with an "Accept / Ignore" button. Accept → change hits the excel files. Ignore → nothing happens.

  1. Upgrade your bot to emit recommendations
    Copy bots/bizbotbash_champion.py from the repo. Modify decide() to return actions as suggestions with rationale instead of executing directly: {"action": "issue_po", "product": "PRD-014", "qty": 200, "why": "stockout imminent"}.
  2. Add a "Recommendations" panel to your ERP
    Prompt Claude: "Add a right-side panel that calls the bot once per day and lists its suggested actions. Each item has Accept / Ignore buttons and a short rationale."
  3. Wire Accept → real write
    When the user clicks Accept, reuse the write functions from Stage 02 to modify the xlsx files. When they click Ignore, log it so you can measure how often you disagreed with the bot.
  4. Measure the copilot's accuracy
    After a month of play, ask Claude: "Compare my accept rate by action type. Which recommendations am I accepting more than 80% of the time? Those are candidates for auto-execution in Stage 04."
Why this matters   Real enterprises don't leap from spreadsheets to full autonomy. The copilot stage is where trust is built. It's also where you discover your bot's blind spots — on your terms, with a human safety net.
// End of Stage 03
You have an AI assistant inside your ERP. You see what it wants to do and why. You approve or reject. The approved actions hit the excel files. You've also got a log of your agreements and disagreements — rich data for when you're ready to cut the cord.
04
Stage · Autonomy

Hit Run All.

Five years of business in 30 seconds

Stage 04 is the mic drop. Click one button. Watch your bot run the company for five years, no human in the loop.

You hand control to your bot. It makes every decision: POs, transfers, shelves, discounts. The BizBotBash backend runs the 60-month simulation. At the end you get a final P&L, a complete excel archive, and a report comparing your bot to the baseline.

[Run All] │ ▼ ┌───────────────────────┐ │ BizBotBash backend │ ◀──── your bot is called for every day │ 60 months × ~30 days │ ◀──── ~1800 decisions made autonomously │ ≈ 30 seconds wall │ └───────────┬───────────┘ ▼ Final report + full excel archive
  1. Flip the bot to autonomous mode
    Remove the "suggest, don't execute" wrapper from Stage 03. Your bot's decide() now returns actions directly. No approval loop.
  2. Upload the bot to BizBotBash
    In the dashboard's BizBot Bash tab, hit Submit Bot. Paste your bot code. The server sandboxes it and registers it. Ready to run.
  3. Hit Run All
    Select your bot in the Mode dropdown, set duration to 60 months, click Run Simulation. The backend runs the full 5-year simulation end-to-end.
  4. Review the autopsy
    Compare against Demo Baseline, Champion, and Ai Genius. Where did your bot beat them? Where did it lose? Iterate. Resubmit. Climb the leaderboard.
What you've proven   You took a business simulation, built a data pipeline, shipped an ERP, added a copilot, and then cut the human out entirely. That's the full arc of "AI for business" — not a slide, not a pitch deck, but a working system you built in a weekend.
// End of Stage 04
You have a fully autonomous bot that runs a 5-year simulated business in 30 seconds. You know exactly how it performs, what it decides, and where it's weak. You understand AI-in-business not as a concept but as a system you built with your own hands.
